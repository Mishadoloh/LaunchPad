# LaunchPad

LaunchPad is a product studio workspace for planning and tracking client delivery work. It combines project management, task execution, team workload, operational notes, risks, milestones, time tracking, and dashboard analytics in one application.

The project is structured as a TypeScript monorepo with a React frontend, an Express API, Prisma ORM, SQLite storage, and shared validation contracts.

## What It Does

- Manages clients, projects, milestones, tasks, risks, notes, and time entries.
- Provides a dashboard with delivery metrics, pipeline value, task status, team load, and activity history.
- Includes a kanban-style task board with status updates.
- Supports JWT authentication and role-aware backend routes.
- Uses Prisma models for persistent relational data.
- Shares Zod schemas and TypeScript types between frontend and backend.
- Ships with seed data so the local workspace is useful immediately after setup.

## Tech Stack

- **Frontend:** React, Vite, TypeScript, TanStack Query, Axios, Lucide icons
- **Backend:** Node.js, Express, TypeScript
- **Database:** SQLite with Prisma ORM
- **Validation:** Zod shared through `packages/shared`
- **Auth:** JWT with bcrypt password hashing
- **Tooling:** npm workspaces, TypeScript project builds

## Project Structure

```text
backend/
  prisma/       Prisma schema and seed data
  src/          Express server, routes, middleware, auth, Prisma client

frontend/
  src/          React application, pages, components, API client, styles
  dist/         Production build output generated by Vite

packages/
  shared/       Shared Zod schemas, DTOs, and TypeScript domain types
```

## Getting Started

### 1. Install Dependencies

```bash
npm install
```

### 2. Create Environment Files

On Windows PowerShell:

```powershell
Copy-Item .env.example backend/.env
Copy-Item .env.example frontend/.env.local
```

On macOS or Linux:

```bash
cp .env.example backend/.env
cp .env.example frontend/.env.local
```

### 3. Prepare the Database

```bash
npm run db:generate
npm run db:push
npm run db:seed
```

### 4. Start the App

```bash
npm run dev
```

Local URLs:

- Frontend: http://localhost:5174
- API: http://localhost:4200
- Health check: http://localhost:4200/api/health

Local account:

- Email: `admin@launchpad.dev`
- Password: `launchpad123`

## Scripts

```bash
npm run dev          # Start frontend and backend in development mode
npm run build        # Build shared package, backend, and frontend
npm run typecheck    # Run TypeScript checks across the workspace
npm run db:generate  # Generate Prisma client
npm run db:push      # Sync Prisma schema with local SQLite database
npm run db:seed      # Seed local workspace data
npm run db:reset     # Reset the local database
```

## Deployment Notes

The frontend and backend can be deployed independently.

For a static frontend deployment, build the frontend with:

```bash
npm run build -w @launchpad/web
```

The generated files will be in:

```text
frontend/dist
```

If the frontend is hosted separately from the API, set:

```text
VITE_API_URL=https://your-api-domain.com/api
```

The backend should be deployed as a Node.js service with access to a database and these environment variables:

```text
DATABASE_URL=
JWT_SECRET=
PORT=
WEB_ORIGIN=
```

## Notes for Reviewers

LaunchPad is organized to make the system easy to inspect:

- Shared validation prevents the frontend and backend from drifting apart.
- Routes are split by domain instead of being placed in one large server file.
- Prisma keeps the data model explicit and easy to evolve.
- The frontend is divided into reusable components, pages, API helpers, and context providers.
